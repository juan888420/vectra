# Glosario de dominio — Vectra

Términos de negocio a usar de forma consistente en código, documentación y commits. Ver también el modelo de entidades en [architecture/data-model.md](architecture/data-model.md).

| Término                                            | Definición                                                                                                                                                                                                                                                    |
| -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Account** (cuenta)                               | Agrupador de transacciones dentro del sistema del usuario (ej. efectivo, cuenta bancaria, tarjeta). No es una cuenta bancaria real conectada, es una representación manual dentro de Vectra.                                                                  |
| **Transaction** (transacción)                      | Un movimiento individual de dinero: un gasto o un ingreso, con monto, fecha, cuenta y categoría.                                                                                                                                                              |
| **Category** (categoría)                           | Clasificación temática de una transacción (ej. Comida, Transporte, Salario). Una categoría es de gasto o de ingreso, nunca ambos.                                                                                                                             |
| **Budget** (presupuesto)                           | Límite de gasto definido por el usuario para una categoría, en un período determinado (ej. mensual).                                                                                                                                                          |
| **Budget period** (período de presupuesto)         | Ventana de tiempo sobre la que se evalúa un presupuesto (ej. un mes calendario).                                                                                                                                                                              |
| **Recurring transaction** (transacción recurrente) | Plantilla que genera transacciones automáticamente cada cierta frecuencia (ej. salario mensual, suscripción semanal/mensual).                                                                                                                                 |
| **Saving goal** (meta de ahorro)                   | Objetivo de ahorro con un monto meta y un monto acumulado actual, independiente del ledger de gasto/ingreso diario. Diseñada en el modelo de dominio; implementación programada para la Fase 3 (Expansión) del [roadmap](product/roadmap.md), no para el MVP. |
| **Net worth** (patrimonio neto)                    | Suma de los balances de todas las cuentas del usuario en un momento dado. No confundir con "gasto del período".                                                                                                                                               |
| **Balance**                                        | Monto actual disponible en una cuenta, derivado de sus transacciones (no almacenado como fuente de verdad independiente).                                                                                                                                     |
| **Money**                                          | Value object que combina un monto y una moneda (`Currency`); encapsula la aritmética monetaria y evita mezclar monedas distintas sin conversión explícita. Nunca se representa con `float`.                                                                   |
| **Currency** (moneda)                              | Value object para el código de moneda (ISO 4217) y su precisión de unidad menor. El MVP asume una sola moneda por usuario; el multi-moneda es un ítem de la Fase 3 del roadmap.                                                                               |
| **Date range** (rango de fechas)                   | Value object con fecha de inicio y fin, usado por períodos de presupuesto y reportes históricos.                                                                                                                                                              |
| **Archived** (archivado)                           | Estado de una `Category` o `Account` que ya no puede usarse en transacciones nuevas pero se conserva para no romper el historial. Vectra nunca borra físicamente una entidad con transacciones asociadas.                                                     |
| **MVP**                                            | Alcance mínimo viable definido en [product/vision.md](product/vision.md), Fase 1 del [roadmap](product/roadmap.md).                                                                                                                                           |
